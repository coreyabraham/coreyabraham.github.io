Project Details

Game Engine: Unity

Platform: PC // Windows 7+ x64

Role(s): Systems and UI Programmer

Development Date: September the 8th, 2023 - December the 21st, 2023

Brief

Lights Out is a 3D top-down perspective Horror / Spooky Puzzle Game that was developed as part of the A.I.E Major Production Project for 1st Year Students! It adapts the common fear of the dark during childhood and centers it's game ideas, art-style and game design to fit that mold. This project had a time frame of around 5 weeks of pre-production and 6 weeks of production, creating a small game within that time-frame and making sure it properly functions for release.

Contributions

As this was the first team-based project I had ever worked on in any "professional" capacity (that being with industry tools like Unity of course), I had a lot of learning to do and thus this project doesn't contain as much on my part unlike the other projects seen on this portfolio.

I do, however, still have talking points with this project, so I'll be discussing them now. The main take-away from this project was mainly it's backend systems and UI intergration, as at the time I was still relatively new to the Unity Engine, so developing UI and Backend Systems was a lot tricker than I had hoped, the good news is, is that it ended up working fine by the time the project was built.

As the project was being developed, I worked on a fair amount of things that the other two programmers couldn't at the time simply due to time management issues and having to work on other parts of the project simultaneously. At the start of the project, I was essentially responsible for creating the entire Technical Design Document at a base level, then having it be extended by everyone else as the project continued developement, this ended up saving us a lot of time in the long-run when it came to the Alpha stages of development as we could spend less time on the Pre-Production documentation side and more on the real prototyping of the game itself, which ended up giving us a chance to implement a lot of systematic functionality that otherwise wouldn't have been easy to introduce until much later into the project's development.

I did state earlier I also worked on the UI Programming for the game, a large majority of the code produced for it was my own doing, mainly the Settings JSON and Unity API interfacing functionality. It took me quite a lot of time to get working within the limited time-frame we had to develop the project and a lot of pressure was built up in my head while developing it at the time as a result of it being the first team-based project I worked on of this caliber, however, after showing the work to the rest of the team, they luckily decided to keep it, so it ended up staying in the final release of the game.

I cannot say the same for the Pause Menu however, as that had to be cut during the final moments of the game's production mainly due to how busted it ended up being later on in the project's lifespan. It had a history of being busted during the project, then fixed, then busted again until we ultimately decided to scrap it all together due to it not really providing much of a benefit considering it's short runtime.

Aside from those, I also worked on some Programming logic you see regularly throughout the game, including the Glowtoy object the main protagonist, Ruben, carries around throughout the game. This was originally going to be Programmed by Devlyn, the Lead Programmer for the project, however, I ultimately ended up working on it as a result of Devlyn being very busy with the game's internals and other more complex C# code. Making this Script was when Unity's Programming APIs finally started to make sense in my head, a sort of "light spark" moment to so say, as I finally came to terms with how Unity handles it's rendering and timing of Update methods. Sure the Script could still have been improved, however, I was just mainly happy that it even worked, let alone this well considering how little I understood making measured time via Update loops running on a per frame basis worked, let alone how to do it consistently with no issues. Ever since this project's conlusion, that very tiny knowledge has proved itself so much more than I would've expected otherwise, so I have this project to thank for giving me that understanding I desperately needed.
